# TRP Examples

These are real JSON artifacts generated by the TRP library. You can inspect them, feed them to the verifier, or use them as test vectors.

- `fibonacci_receipt.json` — A ToolReceipt for compute_fibonacci(10)=55, deterministic, replay-class STRONG
- `structured_claim.json` — A StructuredClaim asserting fibonacci(10) produces the expected output
- `compound_claim.json` — A compound AND claim: fibonacci(10)=55 AND fibonacci(10).result > 50
- `verification_result.json` — The result of replaying and verifying the receipt
- `claim_match_result.json` — The result of matching the structured claim against the receipt evidence
- `false_match_result.json` — A FALSE match result: claim expects result=99 but receipt has result=55
- `evidence_bundle.json` — An EvidenceBundle containing the fibonacci receipt plus a document hash
- `remote_tool_receipt.json` — A ToolReceipt for a remote, non-deterministic LLM tool (model_based, no replay)
